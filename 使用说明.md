# OpenCareEyes v0.3 使用说明

## 1. 适用范围

OpenCareEyes 是面向 Windows 10/11 办公与学习用户的本地护眼辅助工具。它可以调节夜间色温和屏幕明暗、提醒规律休息，并提供低打扰的专注与自动化功能。

本软件不是医疗器械，不诊断、治疗或预防眼病，也不承诺“减少蓝光伤害”或“保护视网膜”。如果持续出现眼痛、视力变化或其他异常，请咨询合格的眼科专业人员。

## 2. 安装与首次启动

### 安装版

1. 从项目 [Releases](https://github.com/Odyphus/OpenCareEyes/releases) 下载 `OpenCareEyes_Setup_<version>.exe` 和 `SHA256SUMS.txt`。
2. 使用 `Get-FileHash <文件> -Algorithm SHA256` 核对校验值。
3. 运行安装程序，按需选择桌面快捷方式和开机自启。

### 便携版

下载 `OpenCareEyes.exe` 后直接运行。请勿在压缩包内运行，也不要把文件放在需要管理员权限才能写入的目录。

首次启动会显示三步欢迎流程：

1. 选择办公、阅读、夜间或游戏显示方案。
2. 选择 20-20-20、番茄钟或自定义休息节奏。
3. 配置固定时间/日出日落自动化与开机自启。

日出日落规则要求先选择城市或手动输入位置；程序不会静默使用某个默认城市。设置完成后，主界面会显示下一次自动动作。

## 3. 日常操作

OpenCareEyes 关闭主窗口后仍驻留系统托盘。

- 左键托盘图标：显示或隐藏主窗口。
- 右键托盘图标：打开快速菜单。
- 再次运行程序：唤起现有窗口，不会启动第二个实例。
- 退出：在托盘菜单选择“退出”，确保恢复显示效果并保存设置。

默认热键如下，可在“设置 → 热键”中修改、恢复默认并检查冲突：

| 功能 | 默认热键 |
|---|---|
| 显示舒适度 | `Ctrl+Alt+N` |
| 屏幕调暗 | `Ctrl+Alt+D` |
| 休息提醒 | `Ctrl+Alt+B` |
| 专注模式 | `Ctrl+Alt+F` |

如果热键被其他程序占用，OpenCareEyes 会保留原状态并提示冲突；请改用其他组合。

## 4. 页面说明

### 总览

总览显示当前显示方案、下一次休息、专注状态、下一次自动切换和当前情境。所有入口共享同一状态，托盘或热键操作后总览会同步更新；被智能免打扰暂停时会显示原因、恢复条件和“本次场景继续提醒”。

“暂停全部”只临时停用效果，不覆盖原设置：

- 暂停 30 分钟；
- 暂停 1 小时；
- 暂停到下一次自动切换；
- 手动恢复。

### 屏幕舒适度

- 色温同时使用语义描述和 Kelvin 数值。数值越低越暖，6500K 接近自然白色。
- 调暗强度按百分比显示；拖动时实时预览，松开后保存。
- 办公、阅读、夜间和游戏方案只改变色温与调暗，不修改休息或专注设置。
- 多显示器热插拔后会重建效果。退出程序时会尝试恢复各显示器原 Gamma Ramp。

显卡驱动、HDR、远程桌面或部分显示设备可能不支持 Gamma Ramp。失败时应用会显示原因，不会把界面错误标记为“已启用”。

### 休息节奏

- 20-20-20：默认每 20 分钟提醒看向约 20 英尺（约 6 米）外 20 秒。
- 番茄钟：适合固定工作/休息周期。
- 自定义：自行设置工作与休息时长。
- 每次进入休息阶段都会自动显示置顶全屏提醒。普通模式可延后 5、10 或 30 分钟，也可安全结束本次休息。
- “倒计时显示”可选择托盘、隐藏或倒计时桌宠。桌宠可拖动，右上角可关闭；关闭只改变显示方式，不会停止计时，可在本页重新开启。
- 全屏、演示、全屏游戏、空闲、锁屏或睡眠时，智能免打扰会隐藏桌宠和休息遮罩并保留剩余时间；自然休息达到 5 分钟后，返回时从完整工作周期重新开始。

高级设置中的严格休息模式会禁用所有“稍后提醒”入口，但始终保留屏幕上的“安全结束本次休息”和 `Esc` 退出途径。需要演示、会议或临时专注时，可提前暂停提醒而不重置原周期。

### 专注模式

设置背景暗化百分比和专注时长后开始。专注效果不会覆盖任务切换器、UAC 安全桌面或受保护的全屏窗口；遇到显示异常可立即通过托盘或热键关闭。

### 自动化

可选择固定时间或日出日落规则，并限制到指定工作日。启用后会立即计算当前应处于的状态，而不是等到第二天才生效。

手动改变显示方案时，覆盖持续到下一调度边界；总览会显示覆盖状态和下一次动作，例如“今晚 19:12 切换到夜间方案”。

“当前情境”卡片可开关智能免打扰、全屏暂停和自然休息。应用例外只接受小写 EXE 文件名，并可分别暂停休息、专注、色温和调暗。多种原因会叠加，只有全部原因消失后才恢复；锁屏和睡眠的安全处理不受总开关影响。

### 设置

- 主题：亮色、暗色或跟随系统，切换后立即生效。
- 动画：跟随系统、标准或减少动画；减少动画会立即停止桌宠装饰动画并显示最终状态。
- 开机自启：写入当前用户的 Windows 启动项，不需要管理员权限。
- 热键：编辑、恢复默认与冲突检测。
- 位置：只用于本机日出日落计算。
- 诊断：查看本地滚动日志、导出不含窗口标题和位置的诊断包。
- 数据：清理本地设置和日志。

## 5. 隐私与本地数据

- 无账号、无广告、无遥测、无云同步。
- 核心功能离线运行，不会上传窗口标题、位置或使用记录。
- 情境检测只在内存中使用 EXE basename；应用例外仅保存该文件名，不保存窗口标题、完整程序路径或前台应用历史。
- 设置由 Qt `QSettings` 保存在当前用户配置中；日出日落只使用用户主动提供的位置。
- 诊断文件仅在用户主动导出时创建。分享前仍应自行检查内容。

Windows 上的主要配置位置为：

- 应用设置：`HKEY_CURRENT_USER\Software\OpenCareEyes\OpenCareEyes`
- 开机自启：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 中的 `OpenCareEyes`

## 6. 卸载与彻底清理

安装版可在 Windows“已安装的应用”中卸载；便携版先从托盘退出，再删除 `OpenCareEyes.exe`。安装器会删除程序文件，并删除由安装任务创建的开机启动项。

为避免升级时丢失偏好，卸载默认保留当前用户设置。需要彻底清理时，优先在“设置 → 数据”中执行清理；若程序已卸载，可在确认路径无误后删除注册表中的 `HKEY_CURRENT_USER\Software\OpenCareEyes`。编辑注册表前建议导出备份。

## 7. 常见问题

### 调节色温后没有变化

先查看总览中的显示器能力和错误提示。HDR、远程桌面、虚拟显示器或显卡驱动可能拒绝 Gamma Ramp。可以临时关闭 Windows 夜间模式以排查叠加效果，但不必永久关闭系统功能。

### 色温和 Windows 夜间模式会冲突吗

两者可能叠加，导致画面比预期更暖。建议只保留一个自动调度来源，避免状态难以判断。

### 提醒没有按预期出现

检查总览、自动化页或托盘是否显示全局暂停、延后或智能免打扰原因。全屏/应用切换结束后有 2 秒稳定期；也可在非锁屏/睡眠场景选择“本次场景继续提醒”。定时器使用单调时钟，系统时间变化不应缩短工作周期。

### 为什么显示“情境检测不可用”

Windows 接口短暂失败时会保留最后状态最多 5 秒；仍无法读取时会解除非安全抑制，避免提醒永久卡住。锁屏和睡眠的安全让出仍然保留。可导出诊断信息后提交 Issue，但请勿粘贴私人路径。

### 主窗口不见了

检查任务栏右下角的隐藏图标，左键 OpenCareEyes 托盘图标；也可再次运行程序唤起已有窗口。

### 如何反馈问题

在 [GitHub Issues](https://github.com/Odyphus/OpenCareEyes/issues/new/choose) 选择对应模板，写明 Windows 版本、显示器数量/DPI、操作步骤和实际提示。安全漏洞请按 [SECURITY.md](SECURITY.md) 私下报告。
