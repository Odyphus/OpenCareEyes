# OpenCareEyes v0.4.1 使用说明

## 1. 适用范围

OpenCareEyes 是面向 Windows 10/11 办公与学习用户的本地护眼辅助工具。它可以调节夜间色温和屏幕明暗、按活跃使用时间提醒休息，并提供低打扰的专注、情境感知和自动化功能。

本软件不是医疗器械，不诊断、治疗或预防眼病，也不承诺“减少蓝光伤害”或“保护视网膜”。如果持续出现眼痛、视力变化或其他异常，请咨询合格的眼科专业人员。

## 2. 安装、升级与首次启动

### 安装版

1. 从项目 [Releases](https://github.com/Odyphus/OpenCareEyes/releases) 下载 `OpenCareEyes_Setup_<version>.exe` 和 `SHA256SUMS.txt`。
2. 使用 `Get-FileHash <文件> -Algorithm SHA256` 核对校验值。
3. 运行安装程序，按需选择桌面快捷方式和开机自启。

### 便携版

下载 `OpenCareEyes.exe` 后直接运行。请勿在压缩包内运行，也不要把文件放在需要管理员权限才能写入的目录。

首次启动会显示三步欢迎流程：

1. 选择办公、阅读、夜间或游戏显示方案。
2. 选择 20-20-20、番茄钟、平衡节奏或自定义休息；新安装默认采用温和渐进提醒并显示桌宠。
3. 配置固定时间/日出日落自动化与开机自启。

日出日落规则要求先选择城市或手动输入位置；程序不会静默使用某个默认城市。设置完成后，主界面会显示下一次自动动作。

### 从 v0.4.0 升级到 v0.4.1

v0.4.1 不改变配置 schema 和既有偏好。它修复全屏休息遮罩被自身 `QUNS_BUSY` 误暂停而闪烁的问题，恢复稳定的逐秒倒计时；点击“结束本次休息”或按 `Esc` 会立即安全退出，且不受屏幕效果或设置同步失败影响。

### 从 v0.3 升级

schema v4 会在迁移前备份设置，写入或同步失败时回滚。升级不会改写已保存的休息开关、严格模式或显示偏好：

- v0.3 用户缺少 `reminder_style` 时迁移为 `fullscreen`，保留原全屏提醒习惯；全新安装才默认 `progressive`。
- 旧休息周期迁入短周期，长周期默认关闭。
- 固定时间调度保留原星期；旧日出日落调度迁移为全周，以保留此前每天执行的实际行为。
- 读取到高于 v4 的配置时以只读方式启动，避免旧程序覆盖新设置。

## 3. 日常操作

OpenCareEyes 关闭主窗口后仍驻留系统托盘。

- 左键托盘图标：显示或隐藏主窗口。
- 右键托盘图标：打开快速菜单。
- 托盘一级“显示倒计时桌宠”：勾选或隐藏桌宠，不改变休息计时。
- 托盘“预览桌宠”与“重置桌宠位置”：检查显示或将不可见的桌宠收回主屏右下角。
- 再次运行程序：唤起现有窗口，不会启动第二个实例。
- 退出：在托盘菜单选择“退出”，确保恢复显示效果并保存设置。

默认热键如下，可在“设置 → 热键”中使用组合键捕获控件修改或恢复默认：

| 功能 | 默认热键 |
|---|---|
| 显示舒适度 | `Ctrl+Alt+N` |
| 屏幕调暗 | `Ctrl+Alt+D` |
| 休息提醒 | `Ctrl+Alt+B` |
| 专注模式 | `Ctrl+Alt+F` |

热键使用 Windows `RegisterHotKey` 注册。保存时会原子替换整组映射：任一组合被占用或无效，程序会恢复原来的全部热键并显示中文错误，不会留下半套新旧混合状态。

## 4. 页面说明

### 总览

总览的“当前实际效果”显示色温、调暗、休息和专注的真实运行状态，并区分：

- 用户已关闭；
- 已实际应用；
- 正在处理；
- 因全局暂停、HDR 或当前情境受抑制；
- 原生操作失败或能力未完全验证。

页面、托盘、热键、方案、调度、全局暂停和情境感知都经过同一效果入口。异步显示命令被接受不代表已经生效；最终以“当前实际效果”和失败提示为准。

“暂停全部”只临时停用效果，不覆盖原设置：

- 暂停 30 分钟；
- 暂停 1 小时；
- 暂停到下一次自动切换；
- 手动恢复。

### 屏幕舒适度

- 色温同时使用语义描述和 Kelvin 数值。数值越低越暖，6500K 接近自然白色。
- 调暗强度按百分比显示；拖动预览最多约 20Hz 合并执行，松开后仅在应用成功时保存。
- 办公、阅读、夜间和游戏方案只改变色温与调暗，不修改休息或专注设置。
- 多显示器热插拔、解锁或睡眠恢复后会重新探测显示能力并重建效果，但 v0.4 不提供每显示器独立方案。

HDR/Advanced Color 开启时，Windows GDI Gamma Ramp 的行为未定义，因此 OpenCareEyes 不调用该接口：

- 色温偏好仍被保留；
- 实际效果显示为“因 HDR 暂停”；
- 可以按界面引导打开 Windows 夜间模式；
- HDR 关闭并重新探测成功后，再按最新偏好应用。

非 HDR 场景下，多输出 Gamma 应用必须全部成功。任一活动输出失败，程序会恢复已修改输出并显示原因，不会把界面错误标记为“已生效”。能力探测暂时不可用时可以进入兼容模式，但会明确标记“未完全验证”。

遇到颜色异常，可使用“恢复原始显示并关闭屏幕效果”。该命令会关闭色温、调暗和专注偏好，再恢复 Gamma 与遮罩，避免调度立即把效果重新应用。

### 休息节奏

v0.4 提供四种节奏：

| 模式 | 短休息 | 长休息 |
|---|---|---|
| 20-20-20 | 活跃 20 分钟后休息 20 秒 | 关闭 |
| 番茄钟 | 活跃 25 分钟后休息 5 分钟 | 关闭 |
| 平衡节奏 | 活跃 20 分钟后微休息 20 秒 | 累计活跃 60 分钟后休息 5 分钟 |
| 自定义 | 独立设置周期和时长 | 可选；长周期必须大于短周期 |

计时遵循以下规则：

- 只累计未暂停且未被情境抑制的活跃时间。
- 空闲 2 到不足 5 分钟时冻结剩余时间；达到 5 分钟视为自然休息并重置短、长周期。
- 全屏、演示、游戏、锁屏和睡眠期间不弹提醒。
- 完成短休息只重置短周期；长休息或自然休息重置两个周期。
- 短、长同时到期时优先长休息。
- 跳过会重置当前到期周期；延后不增加活跃时间。
- 不保存每日或逐应用历史，程序重启后从新周期开始。

### 提醒方式

**渐进提醒（新安装默认）**

1. 到点后，桌宠在右下角展开为互动卡片，不抢键盘焦点。
2. 可以选择“现在休息”“稍后 5/10/30 分钟”或“本次跳过”。
3. 60 秒未处理时升级为更醒目的非阻塞提示；仍可保存文件、切换窗口。
4. 只有点击“现在休息”后才开始休息倒计时。
5. 关闭提示或按 `Esc` 等同于延后 5 分钟，并显示可撤销提示。

**全屏提醒**

保持 v0.3 的即时全屏行为。升级用户默认保留此方式；普通模式仍可按界面提供的入口延后或安全结束。

**严格休息**

到点后立即全屏，并隐藏延后选项。严格模式只限制延后，不取消安全边界：始终保留明确的安全退出按钮和 `Esc`。严格休息仍接受智能免打扰；关闭智能免打扰后，全屏场景不再自动暂停严格休息，但锁屏和睡眠的安全处理始终保留。

### 倒计时桌宠

桌宠是功能型伙伴，用于显示工作/休息/暂停状态、展开提醒和一键开始休息；不包含等级、积分、打卡或养成系统。

- 可从托盘一级菜单或“休息节奏”卡片显示/隐藏。
- 可拖动；位置按逻辑坐标保存，重启后恢复。
- 显示器拔出、分辨率或缩放变化导致位置不可见时，自动回收到主屏右下角。
- 隐藏或被情境抑制后停止桌宠动画和定时器。
- “跟随系统”动画模式会遵循 Windows 客户区动画设置；“减少动画”会立即显示最终帧。

### 专注模式

设置背景暗化百分比和专注时长后开始。专注效果不会覆盖任务切换器、UAC 安全桌面或受保护的全屏窗口；遇到遮罩创建或 Z 序异常时会显示失败，不会把界面标成已启用。

### 自动化

可选择固定时间或日出日落规则，并为日间、夜间分别指定显示方案。启用后会立即计算当前应处于的状态，而不是等到第二天才生效。

- 固定时间：设置切换时间和星期规则。
- 日出日落：设置位置、日间/夜间方案、星期规则，以及日出和日落各 `-120` 到 `+120` 分钟偏移。
- 手动改变显示方案：覆盖持续到下一调度边界；总览会显示覆盖状态和下一次动作，例如“今晚 19:12 切换到夜间方案”。

“当前情境”卡片可开关智能免打扰、全屏暂停和自然休息。应用例外表头分别为“暂停休息/隐藏专注/暂停色温/暂停调暗”。选择 EXE 时，UI 会立即提取小写文件名；完整路径不会进入 Controller、设置、日志或诊断包。

多种抑制原因会叠加，只有全部原因消失后才恢复。用户在抑制期间关闭功能或调度切换方案，恢复时会读取最新偏好，不恢复旧快照。情境传感器连续失败 5 秒后解除 idle、全屏和应用规则等非安全抑制；锁屏和睡眠仍保持安全暂停。

### 设置

- 主题：亮色、暗色或跟随系统，切换后立即生效。
- 动画：跟随系统、标准或减少动画。
- 开机自启：写入当前用户的 Windows 启动项，不需要管理员权限。
- 热键：组合键捕获、恢复默认、冲突检测与原子批量保存。
- 位置：只用于本机日出日落计算。
- 检查更新：仅在点击后请求 GitHub 最新 Release，5 秒超时；不后台下载或安装。
- 诊断：查看本地滚动日志、导出不含窗口标题、完整 EXE 路径和位置的诊断包。
- 数据：清理本地设置。滚动日志独立保存在 `%LOCALAPPDATA%\OpenCareEyes\logs`，可在退出程序后手动删除。

## 5. 更新、网络与隐私

- 无账号、广告、遥测、云同步和持久化使用统计。
- 核心功能离线运行，不会上传窗口标题、位置或使用记录。
- 情境检测只在内存中使用 EXE basename；应用例外仅保存该文件名，不保存完整程序路径或前台应用历史。
- 设置由 Qt `QSettings` 保存在当前用户配置中；日出日落只使用用户主动提供的位置。
- 诊断文件仅在用户主动导出时创建。分享前仍应自行检查内容。
- 程序启动时不检查更新。只有点击“检查更新”才访问 GitHub `releases/latest`，不发送设备标识，不后台下载或自动安装。

Windows 上的主要配置位置为：

- 应用设置：`HKEY_CURRENT_USER\Software\OpenCareEyes\OpenCareEyes`
- 开机自启：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 中的 `OpenCareEyes`

## 6. 卸载与本地数据清理

安装版可在 Windows“已安装的应用”中卸载；便携版先从托盘退出，再删除 `OpenCareEyes.exe`。安装器会删除程序文件，并删除由安装任务创建的开机启动项。

为避免升级时丢失偏好，卸载默认保留当前用户设置和滚动日志。“设置 → 数据”只清理设置，不删除正在使用的日志。需要清除全部本地数据时，请先从托盘退出程序，再在确认路径无误后删除注册表中的 `HKEY_CURRENT_USER\Software\OpenCareEyes` 和 `%LOCALAPPDATA%\OpenCareEyes\logs`。编辑注册表前建议导出备份。

## 7. 常见问题

### 调节色温后没有变化

先查看“当前实际效果”和显示健康状态。HDR 开启时程序会主动暂停 Gamma 色温，并引导使用 Windows 夜间模式；远程桌面、虚拟显示器或显卡驱动也可能拒绝或覆盖 Gamma Ramp。若显示“未完全验证”，可重新探测或使用兼容模式，但不要把它视为已确认生效。

### 色温和 Windows 夜间模式会冲突吗

在 SDR 下两者可能叠加，导致画面比预期更暖。建议只保留一个自动调度来源。HDR 下 OpenCareEyes 不调用 Gamma Ramp，优先使用 Windows 夜间模式。

### 提醒没有按预期出现

检查总览、自动化页或托盘是否显示全局暂停、延后、渐进提示待处理或智能免打扰原因。渐进模式到点后不会直接开始休息倒计时，需在提示卡点击“现在休息”。全屏/应用切换结束后有稳定恢复期；非锁屏/睡眠场景可选择“本次场景继续提醒”。

### 为什么桌宠不见了

先勾选托盘一级“显示倒计时桌宠”。若显示器布局变化，可用“预览桌宠”或“重置桌宠位置”。全屏、演示、游戏、锁屏、睡眠或休息被智能免打扰暂停时，桌宠会暂时隐藏并停止动画。

### 为什么显示“情境检测不可用”

Windows 接口短暂失败时会保留最后状态最多 5 秒；仍无法读取时会解除非安全抑制，避免提醒永久卡住。锁屏和睡眠的安全让出仍然保留。可导出诊断信息后提交 Issue，但请勿粘贴私人路径。

### 检查更新失败会影响核心功能吗

不会。检查更新是用户主动触发、独立且可超时的网络操作；离线、超时、GitHub 限流或无效响应只会显示检查失败，不影响本地功能。程序不会自动下载或安装更新。

### WinGet 能消除 SmartScreen 吗

不能。WinGet 清单和 SHA-256 用于来源与完整性验证，不等同于付费代码签名，也不能保证消除 SmartScreen。仓库中的候选清单只有在 Release 资产固定后通过 `winget validate`、Windows Sandbox 静默安装/升级/卸载测试并提交社区仓库，才可能被官方源收录。

### 主窗口不见了

检查任务栏右下角的隐藏图标，左键 OpenCareEyes 托盘图标；也可再次运行程序唤起已有窗口。

### 如何反馈问题

在 [GitHub Issues](https://github.com/Odyphus/OpenCareEyes/issues/new/choose) 选择对应模板，写明 Windows 版本、显示器数量/DPI、操作步骤和界面显示的错误代码。安全漏洞请按 [SECURITY.md](SECURITY.md) 私下报告。
