# OpenCareEyes v0.6 设计规范

本文件是“伙伴小屋 × Windows Soft Fluent”界面与宠物表现的实现约束。它描述可验证规则，不替代业务状态机或平台安全规则。

## 主题单一来源

所有顶层窗口从 `ThemeManager` 接收同一个不可变快照：

```python
ThemeSnapshot(
    requested,       # system / light / dark
    resolved,        # light / dark
    high_contrast,
    motion_profile,  # standard / reduced
    brand_accent,
    warm_accent,
    pet_accent,
)
```

- `ThemeManager` 是主题、系统高对比度和减少动画状态的唯一解释者。
- 主窗口、欢迎流程、宠物窗口、快捷气泡、休息界面和快捷工具必须在运行中同步切换，不自行猜测 `system` 是亮色还是暗色。
- 高对比度使用系统 `QPalette` 和清晰边界；不能只换一套近似配色。
- 宠物强调色只用于舞台、选择态或伙伴相关点缀，不覆盖品牌主操作色和警告语义。

## 颜色与视觉 token

| 语义 | token / 默认值 | 使用规则 |
|---|---|---|
| 品牌强调 | `#5B8DEF` | 主操作、选中导航、键盘焦点；不可表示成功 |
| 暖色强调 | `#F2A65A` | 色温、夜间和温暖场景 |
| 默认伙伴强调 | `#65BFA5` | 伙伴舞台和轻量状态点缀 |
| 亮色背景/卡片 | `#F5F7FA` / `#FFFFFF` | 内容层次由背景、轻边框和间距共同表达 |
| 暗色背景/卡片 | `#101319` / `#1A202A` | 避免纯黑大面积背景和发光描边 |
| 圆角 | 8 / 9 / 12 px | 输入控件、按钮、内容卡；不把所有容器做成胶囊 |
| 间距 | 8 px 基线 | 常用 8、12、16、24；同组紧、跨组松 |

默认字体为 `Segoe UI`，中文回退 `Microsoft YaHei UI`。标题、正文、辅助文字使用有限字号层级；状态必须同时提供文字，不能只依赖颜色、宠物表情或动画。

## 布局和响应式

- 主窗口默认 `920×640`，支持 100%–200% DPI。
- 陪伴屋首屏采用非对称舞台，伙伴舞台约占可用宽度 58%，另一侧放主状态、下一休息和主要操作。
- 页面只保留一个主标题和一个主要操作；相邻卡片避免重复同一状态。
- 宽度低于 `640` 逻辑像素时，舞台、状态和操作改为纵向单列；导航切到顶部紧凑形式。
- 输入控件和核心按钮不得被裁切，不以横向滚动作为响应式方案。
- 临时消息使用内容上方的浮层 toast，不推动页面重新排版；错误保持可见直到用户处理或关闭。

## 组件规则

- 使用完整的亮/暗 QSS 覆盖标签页、表格、表头、文本编辑、列表、菜单和滚动条。
- 图标使用统一语义资源，不用 Emoji 充当产品图标。
- 衣橱、宠物和方案选择必须有明确选中态，并可通过键盘到达。
- `StatusPresenter` 统一投影伙伴标题、下一休息、暂停/恢复条件，以及色温、调暗、休息和专注的“期望/实际”状态。
- 托盘只保留高频动作；完整配置回到伙伴小屋。快捷气泡依据上下文显示少量操作，并尊重用户保存的快捷顺序。

## 动效预算

- 动效只用于解释状态变化，不做持续呼吸、无意义漂浮或末十秒闪烁。
- 首次显示和资源切换建议 160–180 ms；减少动画时直接显示最终帧。
- 桌面宠物帧动画最高 20 fps；帧调度基于截止时间，事件循环延迟时跳过过期帧，不补播造成加速。
- 连续位置移动目标上限 30 Hz；光标远离时 2 Hz 采样，进入 180 逻辑像素范围后最高 10 Hz。
- `motion_mode=system` 同时遵循 Windows 客户区动画和节电模式；系统减少动画、高对比度安全场景或显式 `reduced` 时使用静态最终帧。
- 宠物隐藏、锁屏、睡眠、全屏抑制或休息遮罩接管时，停止非必要动画、光标和系统指标采样。

## 差分渲染与性能

- `AppState` 只在语义变化时发布；倒计时、动画帧和指标使用专用 tick/展示信号。
- `CompanionPresentationSnapshot` 只携带绘制伙伴所需的稳定字段。相同快照不得重复触发 `setFixedSize()`、`setStyleSheet()`、`show()`、`raise()` 或资源重载。
- 窗口避让必须先判断是否允许移动，再访问原生窗口几何；禁用、隐藏、拖动、休息或安全抑制时停止轮询。
- PNG 解码和 DPR 缩放使用有界缓存；重复图集帧共享源图，不逐帧重复解码。
- 稳定空闲目标：CPU 低于 1%、工作集低于 150 MiB、启动不超过 3 秒。隐藏或静态伙伴不得保留活动动画定时器。

## 宠物包 schema v2

官方宠物包仍是无脚本声明式资源：

```text
assets/pets/<pet_id>/
├─ manifest.json
├─ preview.png
├─ sprites/
├─ accessories/
├─ effects/
└─ sounds/
```

v2 在 v1 的独立 PNG 帧基础上增加：

- `asset_scale: 1 | 2`，正式高清资源推荐 `2`；逻辑画布大小仍由 `canvas_size` 定义。
- `PetVisualTheme`：`accent`、`warm_accent`、`stage_light`、`stage_dark`。
- `PetFrame.source_rect = (x, y, width, height)`，允许多个动作共享 PNG atlas。
- 单动作最多 60 帧，单包动作帧总数最多 256；运行时仍限制 20 fps。

兼容规则：运行时接受 schema v1 和 v2；v1 未声明图集、2x 比例或视觉主题时使用默认值。v2 `source_rect` 必须位于真实 PNG 边界内。所有资源仍只允许包内 PNG、WAV 和 JSON，拒绝绝对路径、`..`、符号链接、脚本、网络地址和系统命令。

白鼬正式包使用 192×192 逻辑画布、2x 资源与共享图集。未来宠物必须通过不同画布和动作映射测试，业务代码不能假设宠物有手、脖子或尾巴。

## 无障碍

- 正文和关键状态目标对比度至少 4.5:1。
- 纯键盘可完成宠物选择、核心设置、立即休息、暂停/恢复和安全退出。
- 每个交互控件提供可访问名称；焦点框在亮色、暗色和高对比度下都清晰可见。
- 200% DPI 不裁切；状态同时使用文字，不只依赖色点、配饰、声音或动作。
- 动画和声音可分别关闭；整点声音默认关闭，并遵守安静时段和安全情境。

## 视觉资源验收

- 透明 PNG 边缘无黑边、白边或色键残留；同一宠物各帧比例、视角和锚点一致。
- 2x atlas 的每个 `source_rect` 必须通过构建期边界检查；预览图必须来自实际随包资源。
- 新增资产需记录作者和许可，进入安装包的第三方资源同步更新 `THIRD_PARTY_NOTICES.md` 与许可文本。
