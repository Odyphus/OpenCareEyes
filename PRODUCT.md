# OpenCareEyes 产品说明

## 产品定位

OpenCareEyes 是一个以桌面伙伴为主体的 Windows 学习、办公与休息陪伴助手。桌面宠物是用户每天看见和操作的入口；色温、调暗、活动加权休息、专注、自动日程和快捷工具是伙伴提供的基础能力，而不是彼此割裂的工具集合。

当前第一只官方伙伴是白鼬“鼬鼬”。底层采用通用宠物引擎和声明式官方宠物包，未来增加其他物种时，不应修改休息、天气、显示效果或工具业务逻辑。

产品继续保留 `OpenCareEyes` 名称、安装标识和升级路径，副标题为“Windows 桌面陪伴与护眼助手”。

## 面向用户

- 长时间使用 Windows 学习、写作、编程或办公的人。
- 希望获得低打扰休息提醒，但不希望注册账号或上传使用数据的人。
- 喜欢桌面角色陪伴，同时要求工具稳定、可关闭且不妨碍全屏或工作窗口的人。

## 核心体验原则

1. **伙伴优先**：日常操作先经过宠物、快捷气泡和托盘，完整设置进入“伙伴小屋”。
2. **真实可信**：界面区分用户偏好和实际效果；HDR、原生调用失败或情境抑制不能显示成“已生效”。
3. **低打扰**：全屏、演示、游戏、锁屏和睡眠时主动让出；情境结束后按最新偏好恢复。
4. **本地优先**：无账号、无遥测，核心陪伴、休息和护眼功能离线运行。
5. **随时可退**：桌宠、动画、声音和联网天气都可关闭；严格休息仍保留明确按钮与 `Esc` 安全退出。

## 界面信息架构

### 桌面日常层

- `PetSurface`：稳定透明窗口，只负责角色绘制、命中和鼠标互动。
- `PetBubble`：单击伙伴后出现的情境快捷气泡，默认操作可由用户排序。
- 系统托盘：保留显示伙伴、立即休息、暂停全部、打开伙伴小屋和退出等高频入口。
- `RestSurface`：消费唯一休息状态机，显示休息场景、倒计时和安全退出。

### 伙伴小屋

| 页面 | 职责 |
|---|---|
| 陪伴屋 | 约 58% 宽度的实时伙伴舞台、伙伴主状态、下一次休息、主要操作和紧凑工具/状态栏 |
| 宠物图鉴 | 官方宠物选择、预览、性格、大小和衣橱；不承载应用规则 |
| 学习桌 | 专注陪伴与屏幕舒适度；不重复设置总览 |
| 休息角 | 休息节奏、提醒方式和休息场景 |
| 自动日程 | 日夜调度、当前情境、智能免打扰和应用道具/例外规则 |
| 设置 | 主题、动画、声音、自启动、热键、位置、隐私、诊断和更新 |

窗口默认约 `920×640`。宽内容区采用舞台与信息非对称布局；可用宽度低于 `640` 逻辑像素时切换为单列，禁止依靠横向滚动才能完成核心操作。

## 功能边界

### 桌面伙伴

- 同时只运行一只官方内置宠物，可在图鉴中切换。
- 宠物拥有动作、性格、装扮和语义事件映射，但不能改变休息计时或系统安全规则。
- 点击、拖动、光标靠近、休息到点、天气、节日和学习应用可触发表现。
- 用户拖动更新永久锚点；自主移动和窗口避让不覆盖该锚点。

### 陪伴与效率

- 活动加权休息、渐进/全屏/严格提醒和四种休息场景。
- 单个用户倒计时、本地便签、按需 CPU/内存/时间状态和带系统确认的回收站清理。
- 色温、调暗、专注、固定时间或日出日落自动化，以及按 EXE basename 配置的情境例外。
- 可选天气换装。天气默认关闭，首次启用必须先获得明确授权。

## 隐私和网络

- 不创建账号，不收集遥测、广告标识或持久化互动统计。
- 不保存窗口标题、完整程序路径、鼠标轨迹、前台应用历史、天气结果或每日使用历史。
- 应用规则只保存小写 EXE basename；便签正文不进入 `AppState`、日志、备份或诊断包。
- 天气是可选网络能力：授权后只向 Open-Meteo 发送经纬度，结果只保存在内存，不记录含坐标的 URL。
- 更新检查只在用户点击后访问 GitHub，不在启动时检查，不后台下载或安装。

## 医疗边界

OpenCareEyes 不是医疗器械。文案只描述调节夜间色温、改善主观观看舒适度和帮助形成休息习惯，不承诺治疗、减少蓝光伤害或保护视网膜。

## v0.7 非目标

- 第三方宠物包导入、在线宠物商店或宠物脚本。
- 多只宠物同时常驻、等级、积分、连续打卡或养成数值。
- 账号、云同步、遥测、持久化使用统计或 AI 推荐。
- 自动安装更新、每显示器独立方案、DDC/CI 或跨平台支持。
- 根据麦克风、音频或窗口标题判断真实会议通话。

这些边界用于保护稳定性、隐私和产品焦点；未来版本若调整，必须同步更新本文件、设计规范、使用说明和变更记录。
