# Changelog

本项目遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)；版本号遵循语义化版本。

## [Unreleased]

## [0.7.0] - 2026-07-18

### Added

- 新增 `CompanionRuntime`，集中管理桌面伙伴的光标采样、自主行为、位置动画、窗口避让、快捷气泡和状态同步；程序入口只负责依赖装配、单实例和生命周期。
- 将 Controller 的公开命令按显示、休息与专注、自动化、宠物与工具拆分为四类处理器，同时保留原命令接口与信号语义。
- 显示效果事务新增请求 ID 与 `applying → compensating → completed` 阶段；Gamma 应用和补偿等待串行 worker 的完成结果，过期结果不会覆盖较新的请求。
- 设置事务在写入前保存完整快照和待提交标记；启动发现中断提交时恢复旧快照，恢复失败则只读启动，避免继续写入混合状态。

### Changed

- 浮动伙伴提醒与普通休息卡统一提供“现在休息、稍后 5/10/30 分钟、本次跳过”，并直接调用同一休息状态机，不再把到期提醒路由为新的手动周期。
- 自然结束、按钮结束、`Esc`、跳过、关闭提醒、全局暂停或安全情境抑制后，会在同一事件循环撤销休息动作并恢复一次 `idle`，不遗留睡眠动画或休息定时器。
- 完整状态只在语义变化时发布；休息倒计时、宠物动画帧、光标采样和工具倒计时改用专用轻量信号，避免高频重建 `AppState`。
- 休息遮罩、渐进提醒、伙伴气泡、快捷工具、首次向导和撤销提示统一消费 `ThemeSnapshot`；高对比度优先使用系统调色板，窄屏和 200% DPI 布局提供纵向滚动且不依赖横向裁切。
- 减少动画模式会立即停止自主移动、位置动画和非必要定时器，恢复永久锚点并显示静态最终帧；恢复标准动画时只启动一个新周期。
- 鼠标唤起伙伴气泡继续不抢焦点；从托盘或键盘入口打开时可聚焦，并支持 `Tab`、`Enter`、`Esc` 与可见焦点环。
- Windows CI 按提交、定时和标签分层：提交检查 Python 3.10–3.13、测试、静态检查与安装器冒烟；稳定性任务检查资源增长和天气未授权零网络；标签门禁复用同一构建产物，并以实际 v0.4.1 安装包作为升级基线；main 与标签均在稳定性门禁通过后继续发布流程。

### Fixed

- 显示效果应用或补偿失败时，状态会保留请求阶段和失败原因，不再提前显示为已完成。
- 设置同步失败或进程在提交中断后，不再静默保留部分新值；无法可靠恢复时禁止继续写入。

### Compatibility

- 配置 schema 继续保持 v6；v0.6.x 的宠物、位置、装扮、快捷气泡、护眼、休息、专注、日程、天气、声音和热键偏好均按原键读取，不新增迁移步骤。

## [0.6.1] - 2026-07-16

### Fixed

- 修复自主踱步期间光标靠近事件抢占 `move` 动作，导致宠物移动时显示站立或盯光标姿势的问题；移动期间现在会持续播放独立的双帧步态动画。
- 修复光标位于宠物本体上时因跨过中心线而高频左右镜像翻转的问题；朝向现在会在本体范围内保持稳定，并与实际移动方向一致。

## [0.6.0] - 2026-07-16

### Added

- `ThemeManager` 与不可变 `ThemeSnapshot`，统一解析亮色/暗色、Windows 高对比度、系统动画偏好、节电状态、品牌强调色和宠物强调色，并同步到主窗口、欢迎流程、宠物、快捷气泡、休息界面与快捷工具。
- 纯业务 `StatusPresenter`，为伙伴小屋、托盘和气泡统一投影伙伴主状态、下一次休息、全局暂停/情境恢复条件，以及色温、调暗、休息、专注的期望与实际状态。
- 宠物包 schema v2：支持 `PetVisualTheme`、`asset_scale` 和 atlas 帧 `source_rect`；白鼬正式包升级为 192×192 逻辑画布、2x PNG atlas，同时保留 schema v1 加载兼容。
- `CompanionPresentationSnapshot` 与专用展示更新链路，将宠物动作、外观和可见性变化与完整 `AppState` 发布分离。
- schema v6 设置迁移，只新增 `companion/quick_actions_json` 保存快捷气泡顺序，v5 所有既有偏好保持不变。
- [产品说明](PRODUCT.md) 与 [设计规范](DESIGN.md)，固定桌面伙伴定位、伙伴小屋信息架构、隐私/非目标、主题 token、响应式、动效预算、宠物包和无障碍规则。

### Changed

- 陪伴屋改为以实时伙伴舞台为中心的非对称首屏；宽屏舞台约占 58%，内容宽度低于 640 逻辑像素时切换单列和顶部紧凑导航。
- 信息架构收敛：学习桌移除重复总览，应用道具/例外规则移至自动日程，休息角聚焦节奏、提醒和场景，宠物图鉴增加明确的宠物与衣橱选择态。
- 伙伴展示采用差分渲染；相同状态不再重复调整窗口大小、样式、显示层级或重新加载资源。窗口避让先检查是否允许移动，再访问原生几何。
- 桌面帧动画限制为最高 20 fps，并按截止时间跳过过期帧；光标采样在远离与靠近伙伴时自适应为 2 Hz 和 10 Hz。减少动画、节电或安全抑制时停止非必要动画与采样。
- 主窗口临时通知改为不推动内容重排的浮层提示；快捷气泡按照用户保存的操作顺序显示情境入口。

### Fixed

- 修复跟随系统主题时部分独立窗口误用暗色、主题切换后控件样式不同步，以及高对比度仍依赖局部硬编码配色的问题。
- 修复重复完整状态发布造成宠物 `show`/`raise`、缩放、气泡重设主题和资源解码抖动的问题。
- 修复窗口避让即使伙伴不可移动仍读取前台窗口几何，以及伙伴隐藏/受抑制后非必要轮询没有及时停止的问题。

### Accessibility

- 补齐标签页、表格、表头、文本编辑、列表和键盘焦点的亮暗主题状态；200% DPI 与窄窗口下核心操作不依赖横向滚动。
- 状态文字明确同时表达用户偏好、实际效果、失败或抑制原因和恢复条件，不只依赖颜色、配饰、声音或宠物动作。

## [0.5.0] - 2026-07-15

### Added

- 通用官方宠物包规范：宠物以声明式 `manifest.json` 描述动作、事件绑定、性格、外观槽位和资源，只允许包内 PNG、WAV 与 JSON；拒绝绝对路径、路径穿越、脚本、网络地址、系统命令、超大帧和不兼容版本。
- `PetPackRegistry`、`CompanionCoordinator`、`PetAnimator`、稳定透明 `PetSurface` 与独立 `PetBubble`。切换宠物会先验证和预加载，失败时恢复原宠物；默认宠物损坏时使用静态占位角色并显示错误。
- 第一只官方宠物白鼬“鼬鼬”，包含发呆、眨眼、歪头、睡觉、踱步、玩耍、点击、拖动、松手踉跄、右键害羞、光标靠近和休息提醒动作。
- 宠物主导的“陪伴屋、宠物图鉴、学习桌、休息角、自动日程、设置”六页导航；护眼、专注、休息、调度和热键仍复用原有业务状态与效果协调入口。
- 天气服务在用户明确授权后通过 QtNetwork 异步访问 Open-Meteo；天气默认关闭，仅发送经纬度，5 秒超时，每 30 分钟刷新，旧结果最多在内存使用 2 小时。
- 白鼬的晴、雨、雪、雷暴换装，离线圣诞节/元宵节规则，以及按小写 EXE basename 触发的写作、计算器和 EDA 学习道具。
- 眺望远方、雪球呼吸、伸展陪练和团子安睡四种休息场景；宠物表现继续订阅唯一 `BreakReminder` 状态机，不接管倒计时和跳过规则。
- 独立用户倒计时、最多 50 条本地便签、按需 CPU/内存/时间小窗，以及显示项目数/大小并保留 Windows 确认的回收站清理工具。
- 混合 DPI 与负坐标感知的窗口避让：活动显示器稳定 2 秒后迁移，前台窗口遮挡时选择空白角落或边缘探头，恢复后回到永久锚点且不覆盖用户拖动位置。
- 整点 1–12 数字气泡、可选 WAV 音效和安静时段；锁屏、全屏、休息或伙伴隐藏时保持静音，不补播错过的报时。
- 毛线球、热可可和松果互动，以及只允许宠物包已声明资源的手动衣橱；瞬时道具结束后恢复最新的节日、天气或学习场景外观。
- schema v5 配置与 v4→v5 原子迁移，保存当前宠物、显示状态、大小、相对锚点、外观偏好、天气开关和休息场景，同时无损保留护眼、休息、专注、调度与热键设置。
- 使用不同画布尺寸和动作映射的第二个测试宠物包，验证业务代码不依赖白鼬物种或肢体结构。

### Changed

- 产品定位调整为“Windows 桌面陪伴与护眼助手”，保留 OpenCareEyes 应用名、安装标识与升级路径。
- 原倒计时桌宠升级为通用桌面伙伴：单击展开快捷气泡，拖动更新永久锚点，右键播放宠物反应；隐藏或受安全情境抑制后停止动画和光标采样。
- 宠物包切换在同一个 `PetSurface` 窗口内完成 160 ms 淡出和淡入；加载失败恢复原宠物，不改变休息、计时和位置状态。
- v4 已显示桌宠的用户升级后默认显示白鼬；原先隐藏桌宠的用户继续隐藏。原 `pet_x/pet_y` 转换为自由锚点并继续保留兼容坐标；找不到已选宠物时回退白鼬。

### Fixed

- 修复重复点击动作不重播、宠物切换后缩放丢失、淡入期间隐藏导致配饰被清空，以及隐藏预览后定时器短暂残留的问题；动画缓存同时按帧数和 32 MiB 字节上限约束。
- 宠物包加载失败时自动回退到默认白鼬，并保留待恢复宠物标识；资源修复后会在后续启动自动重试，显式切换宠物则清除旧恢复标识。
- 修复天气坐标切换时旧缓存和迟到请求串用、窗口几何或 DPI 映射失败时误恢复位置、混合 DPI 跨屏窗口包围框误判，以及临时避让期间自主移动竞争的问题。
- 修复用户倒计时在截止边界暂停后卡在零秒、无效标签覆盖现有计时器、整点报时配置校验失败后半更新；倒计时运行、暂停和结束状态现在进入统一只读状态，但秒级更新仍只使用专用 tick。

### Security

- 宠物资源现在验证真实 PNG 内容、尺寸和像素总量，不再只信任文件扩展名；诊断导出会清除常见便签字段，天气缓存也严格绑定到对应坐标。

### Privacy

- 天气未授权时保持零天气网络请求；不记录含经纬度的 URL，天气结果、系统指标和鼠标坐标不持久化。
- 宠物互动次数、鼠标轨迹、窗口标题、完整程序路径和前台应用历史均不进入设置、日志、备份或诊断包。
- 便签正文只保存在本地便签文件，不进入 `AppState`、日志或诊断包；系统指标仅在对应小窗打开时采样。

### Limitations

- v0.5.0 同时只运行一只随软件发布的官方宠物，不开放第三方导入、在线商店、多宠物常驻、养成数值或宠物脚本。
- 第二只正式宠物将在后续版本验证“只增加宠物包、不修改业务逻辑”的接入路径；当前第二个不同尺寸宠物仅作为自动化测试夹具，不进入发行版图鉴。

## [0.4.1] - 2026-07-14

### Fixed

- 修复 OpenCareEyes 自己的全屏休息窗口触发 `QUNS_BUSY` 后被智能免打扰误判为外部全屏，导致遮罩周期性闪烁、倒计时冻结和按钮点击中断的问题；外部 busy、演示、D3D 和真实全屏检测保持原行为。
- 休息遮罩仅在显示器联合几何实际变化时调整窗口；普通 `break_tick` 只更新倒计时文字，不再重复显示、隐藏、激活或设置整屏几何。
- “结束本次休息”和 `Esc` 改为纯运行时安全退出，不再受 Gamma、调暗、效果协调或设置同步失败影响，操作后立即返回工作状态并隐藏遮罩。

## [0.4.0] - 2026-07-13

### Added

- `RuntimeIntent`、`ReconcileResult`、`DisplayHealthState`、`BreakCadenceState`、`BreakPromptState`、`UserNotice` 和 `UpdateState`，用于区分用户偏好、实际效果、提示阶段与异步结果。
- HDR/Advanced Color 能力探测：HDR 开启时抑制 Gamma Ramp、保留用户偏好，并在总览和屏幕舒适度页显示原因及 Windows 夜间模式引导。
- 活动加权的 20-20-20、番茄钟、平衡节奏和自定义短/长休息；空闲达到 5 分钟可作为自然休息重置周期。
- 温和渐进提醒：先显示不抢焦点的桌宠互动卡，支持立即休息、延后 5/10/30 分钟和本次跳过；未处理 60 秒后升级为更醒目的非阻塞提示。
- 托盘一级“显示倒计时桌宠”开关，以及桌宠预览、位置保存、重置和超出可见区域自动回收。
- 自动化日间/夜间方案与日出、日落 `-120` 到 `+120` 分钟偏移。
- 仅由用户触发的 GitHub Release 更新检查，5 秒超时；启动和后台运行不发起更新请求，也不自动下载安装。
- `Odyphus.OpenCareEyes` WinGet 候选清单生成与关键字段测试。正式提交仍需在 Release 固定后运行 `winget validate` 并完成 Windows Sandbox 安装、升级和卸载测试。
- 便携版、安装版和 Release 随附 Python、PyInstaller、PySide6/Qt、Astral、darkdetect 的第三方许可与来源说明。
- schema v4 配置迁移、幂等保护、原子备份/回滚及未来 schema 只读启动。

### Changed

- `EffectCoordinator.reconcile(RuntimeIntent)` 成为色温、调暗、休息和专注实际写入的统一入口；页面、托盘、热键、方案、调度、全局暂停和情境感知只更新意图。
- 命令执行统一为“验证 → 设置快照 → 应用效果 → checked sync → 发布状态”；任一步失败时恢复设置和已应用效果，并在当前操作表面显示中文错误。
- 多输出 Gamma 应用改为全成功或全回滚；显示变化、解锁和睡眠恢复后重新探测能力。
- 热键从 `keyboard` Hook 改为 Win32 `RegisterHotKey`；原子批量注册失败时恢复旧组合。
- 完整状态只在语义变化时发布，倒计时通过独立 `break_tick` 更新，避免稳定情境每秒重建完整 `AppState`。
- 全新安装默认使用渐进提醒和桌宠；v0.3 用户缺失 `reminder_style` 时迁移为原全屏行为。
- 固定时间调度保留原星期规则；旧日出日落调度迁移为全周，再正确应用新的星期规则。
- 产品定位继续限定为观看舒适度和休息习惯，不宣称治疗、减少蓝光伤害或保护视网膜。

### Fixed

- 情境抑制、全局暂停、调度与用户操作不再绕过同一效果入口，避免界面与实际效果不一致或恢复旧快照。
- Gamma Ramp 静默失败、部分输出成功、遮罩创建失败或设置同步失败时不再显示“已生效”。
- 传感器连续失败 5 秒后解除 idle、全屏和应用规则等非安全抑制，避免永久卡在暂停；锁屏和睡眠仍保持安全暂停。
- 日出日落自动化现在正确执行星期规则。
- 高对比度、200% DPI、组合键捕获和可访问状态提示得到补强。

### Privacy

- 不保存每日或逐应用休息历史；重启后从新周期开始。
- EXE 选择器在 UI 层立即提取小写 basename，完整路径不得进入 Controller、设置、日志、备份或诊断包。
- 手动更新检查不发送设备标识；除用户点击检查外保持零更新网络请求。

## [0.3.0] - 2026-07-13

### Added

- 智能免打扰：识别全屏窗口、演示、独占 D3D、空闲、锁屏与睡眠情境，并在情境结束后恢复。
- 按小写 EXE 文件名配置的应用例外，可分别暂停休息、专注、色温和调暗效果。
- 自动化页“当前情境”卡片、总览与托盘统一的暂停原因/恢复条件，以及“本次场景继续提醒”。
- schema v3 配置迁移、未来版本只读保护和迁移失败回滚。
- 桌宠淡入、状态过渡与低频眨眼动画，支持系统减少动画、显式标准和精简模式。

### Changed

- `AppController` 保持命令门面职责；状态投影、情境策略和效果补偿拆分到独立协调器。
- 新安装默认使用 20-20-20、`1200/20` 和工作日 `19:00–07:30` 固定计划；升级用户保留旧有效默认。
- 自动抑制只改变实际运行状态，不覆写用户开关、严格休息或调度设置。

### Fixed

- 同一窗口每秒复查不再重启进入/退出防抖，避免全屏暂停或恢复永久等待。
- 情境效果部分失败时回滚并报告，补偿失败也会在界面与日志中可见。
- 休息倒计时边界增加 Windows 定时器提前唤醒容差，消除偶发重复秒数。

## [0.2.1] - 2026-07-12

### Added

- 可拖动、可关闭并能从休息设置重新开启的置顶倒计时桌宠。

### Fixed

- 所有休息阶段现在都会自动显示置顶全屏提醒，不再只有严格模式才出现遮罩。
- 严格休息会从全屏、主页面和托盘统一禁用延后，同时始终保留按钮和 `Esc` 安全退出。

## [0.2.0] - 2026-07-12

### Added

- Windows 11 Soft Fluent 风格的侧边导航、总览页与三步首次使用流程。
- 全局暂停、休息延后、可编辑热键、下一次自动动作及诊断导出。
- 单实例唤起、动态亮暗主题和本地优先的隐私说明。
- Windows CI：pytest、Ruff、PyInstaller onefile 构建和 EXE 启动冒烟测试。
- 标签发布流程：Inno Setup 安装包、SHA-256 清单和自动生成的 Release Notes。

### Changed

- 主界面、托盘、热键和自动调度改为共享应用状态。
- 休息计时改为基于单调时钟的状态机，调度启用后立即计算当前状态。
- 产品文案聚焦观看舒适度和休息习惯，不再宣称减少蓝光伤害或保护视网膜。
- `pyproject.toml` 成为版本号与 Python 依赖的唯一来源；规范源码分支改为 `main`。
- PyInstaller 与 Inno Setup 统一使用 `dist\OpenCareEyes.exe` 单文件产物。

### Fixed

- 修正 `src/` 布局的源码安装命令、包版本不一致、嵌套源码 ZIP 与安装器输入路径。
- 补全 Apache License 2.0 正文和卸载后本地设置的说明。
