# 安全策略

## 支持版本

| 版本 | 安全更新 |
|---|---|
| 0.4.x | 支持 |
| 0.3.x | 仅关键安全修复 |
| 0.2.x | 不支持 |
| 0.1.x 及更早 | 不支持 |

## 私下报告漏洞

请使用仓库的 **Security → Report a vulnerability** 私下提交报告，不要创建公开 Issue。报告请包含受影响版本、Windows 版本、复现步骤、影响和最小必要日志；提交前删除用户名、窗口标题、完整程序路径、位置等个人信息。

维护者会先确认收到报告，再评估影响、准备修复并协调披露。请在修复发布前避免公开可利用细节。

普通功能故障、Gamma/HDR 显示兼容性、`RegisterHotKey` 冲突或手动更新检查超时不属于安全漏洞，请使用 Issue 模板反馈。OpenCareEyes 不会在启动或后台自动检查、下载或安装更新；若观察到非用户触发的更新网络活动，请按安全问题私下报告。
